# Stage 20 Summary

## Milestone

- Stage: 20
- Date: 2026-04-14
- Status: Completed

## Root Cause

- Qualified API call symbols such as `Parameter` and `empty` still did not enter the preprocess candidate pool, so the wordbook remained too narrow on practical deep-learning code.
- The explanation page still buried the actual selected token inside metadata instead of anchoring it visually at the top.
- Model outputs that contained simple markdown constructs such as inline code, lists, or fenced code blocks were still shown as raw text.
- The `Glossary Snapshot` section no longer added enough value to justify the extra visual weight or repeated panel payload.

## What Was Done

- Added qualified dotted-call extraction so callable names from expressions like `nn.Parameter(...)` and `torch.empty(...)` can enter the glossary and wordbook pipeline.
- Added a prominent current-selection focus card at the top of the explanation page.
- Added basic markdown rendering for summaries, sections, wordbook details, and follow-up chat.
- Removed the `Glossary Snapshot` block from the explanation page.
- Stopped sending the now-unused glossary snapshot payload into the webview.

## Key Files

- `src/analysis/glossary.ts`
- `src/extension.ts`
- `src/ui/explanationPanel.ts`
- `src/test/index.test.ts`

## Validation

- `npm.cmd run compile`
- `npm.cmd test`
- Regression coverage confirmed:
  - qualified dotted-call symbols such as `Parameter` and `empty` enter glossary extraction
  - earlier Stage 19 scope-path and preprocess-progress behavior still passes

## Outcome

The extension now feels closer to an actual reading assistant for real-world model code: common library call symbols can reach preprocessing, the page makes the current selection visually obvious, markdown output is far more readable, and the main explanation surface is cleaner after removing the low-value glossary snapshot block.
