# Stage 25 Summary

## Milestone

- Stage: 25
- Name: Panel state resynchronization
- Date: 2026-04-15

## Delivered

- Fixed watched-selection semantics:
  - the panel now counts as "watching selection" only when it is visible
  - hidden-but-still-open panels no longer suppress expected reveal/update behavior
- Fixed panel-ready behavior:
  - when the explanation webview becomes ready, the extension now resyncs glossary and wordbook state
  - if the current source editor already has a non-empty selection, the panel immediately replays an auto explanation for that selection
- Tightened preprocess and wordbook UI syncing:
  - preprocess progress updates now also refresh `currentFile`, visible wordbook entries, and watch state
  - wordbook refreshes now push file context and visible preprocess state together instead of only pushing entries
  - added selection scheduling logs so future regressions in the auto-explain pipeline are easier to diagnose

## Notes

- The bug was not only in rendering. The main issue was that panel visibility, current selection, preprocess progress, and wordbook cache state were drifting apart. This stage closes that gap by making the panel rehydrate itself from the current source-editor state when it becomes ready and by updating file-context fields together with progress and wordbook entries.
