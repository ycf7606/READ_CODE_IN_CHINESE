# Stage 13 Summary

## Milestone

- Stage: 13
- Date: 2026-04-14
- Status: Completed

## Root Cause

- Stage 12 fixed file-scoped preprocessing and stale-task cancellation, but two product gaps remained visible in real use: the preprocess cache was not shown as a first-class file wordbook, and the settings panel still generated the global prompt by local string concatenation instead of using the configured model.
- The panel also still duplicated section content when both bullet items and paragraph content existed, which weakened the intended dictionary-style presentation.

## What Was Done

- Added visible file-wordbook state to the explanation panel and wired it to the current file preprocess cache.
- Refreshed wordbook state on panel ready, editor switch, file save, and preprocess completion.
- Replaced local prompt-template generation in the settings panel with provider-backed prompt-profile generation, while keeping a local fallback path when the remote provider fails.
- Passed the full audience profile and provider hyperparameters into prompt-profile generation so the generated prompt is shaped by real runtime settings.
- Tightened explanation rendering so bullet items are shown as the primary dictionary-style output without duplicated paragraph content.
- Rewrote corrupted local and cache fallback strings into clean concise Chinese.
- Added regression tests for local prompt-profile generation and remote section-item normalization.
- Validated prompt generation and real token explanation behavior against the configured `gpt-5.4` endpoint.

The extension now behaves closer to the intended product surface: users can see the current file wordbook directly in the panel, generated prompts are produced by the configured model instead of a local template, and explanations render as concise dictionary-style bullet points rather than repeated mixed-format text.
