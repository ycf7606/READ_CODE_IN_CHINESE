# Stage 23 Summary

## Milestone

- Stage: 23
- Name: Source editor isolation and auto-explain stabilization
- Date: 2026-04-15

## Delivered

- Stopped non-source editors from hijacking runtime state:
  - output/log editors are now ignored by active-editor tracking
  - the last valid source editor remains the preferred context for panel refresh, glossary refresh, preprocess, and file-level commands
- Stabilized auto-explain request flow:
  - added scheduled, active, and recently-settled signatures for auto explanations
  - blocked duplicate same-selection requests before they can start a new version
  - cleared stale auto-explain state when the real source document changes
- Reduced unnecessary task churn:
  - switching to another view of the same source document no longer cancels explanation or preprocess work
  - saving a tracked source file still refreshes caches even if focus has moved away from the source editor

## Notes

- The root runtime bug was not mainly the provider. The extension state machine was treating VS Code output/log text editors as if they were source editors, which canceled in-flight work and triggered auto-explain against the log buffer itself.
- The provider can still occasionally return `content: null` for preprocess-selection requests, but that path already retries and falls back. This stage fixed the larger event-loop bug that made the feature appear fundamentally broken.
