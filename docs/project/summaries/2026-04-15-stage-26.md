# Stage 26 Summary

## Milestone

- Stage: 26
- Name: Watch/reveal split and wordbook diagnostics
- Date: 2026-04-15

## Delivered

- Restored a safer split between panel-open and panel-visible behavior:
  - selection watching now keys off whether the explanation panel exists
  - automatic panel reveal now keys off whether the panel is currently visible
  - this avoids using one boolean for two different UX decisions
- Added runtime diagnostics around the wordbook cache:
  - logs when a current cache is loaded
  - logs when the cache is missing or outdated
  - logs when preprocess updates and applies wordbook entries
- Kept the Stage 25 panel-ready resync flow in place so the panel still rehydrates the current selection and wordbook context when it starts up

## Notes
