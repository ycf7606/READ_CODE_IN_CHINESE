# Stage 29 Summary

## Milestone

- Stage: 29
- Name: Stage 19 preprocess reliability hardening
- Date: 2026-04-15

## Problem

- The rolled-back Stage 19 baseline can preprocess a file wordbook in batches of 20 symbols.
- Candidate selection already had a remote-to-local fallback.
- Actual batch preprocessing did not.
- Result: if any single remote preprocess batch timed out, returned empty content, or otherwise failed, the whole preprocessing job stopped mid-run.
- This matches the observed user symptom: preprocessing advances partway, then appears to hang or stop.

## Root Cause

- `src/knowledge/symbolPreprocessBuilder.ts` called `options.provider.preprocessSymbols(...)` directly inside the batch loop.
- That loop wrote partial cache after successful batches, but any later thrown batch error escaped the loop and failed the entire run.
- Abort-like errors must still stop immediately, but ordinary provider failures should not discard already completed batches.

## Fix

- Added a local fallback provider inside `buildSymbolPreprocessCache()`.
- Wrapped each batch preprocess request in `try/catch`.
- Preserved current cancellation semantics:
  - abort-like errors are rethrown
  - non-abort remote failures log a warning and fall back only for that batch
- Kept partial cache writes unchanged after each completed batch.
- Marked the overall preprocess source as `mixed` when remote and local batches both contribute.

## Verification

- Added regression coverage for a three-batch run where the second remote batch fails.
- Verified that:
  - preprocessing still completes
  - all entries are written to cache
  - failed-batch entries are filled by the local fallback
  - later batches can still continue
- Validation commands:
  - `cmd /c npm.cmd run compile`
  - `cmd /c npm.cmd test`

## Notes

- This is a reliability fix on top of the Stage 19 baseline, not a reintroduction of the rejected Current Selection-era UI path.
- If the remote provider is flaky, the wordbook may now finish with mixed-quality entries instead of stopping halfway; that is the correct tradeoff for keeping the feature usable.
