# Stage 16 Summary

## Milestone

- Stage: 16
- Date: 2026-04-14
- Status: Completed

## Root Cause

- The earlier wordbook pipeline still depended on local fixed-term filtering before the builder ran, so the configured remote provider never actually participated in preprocess candidate selection.
- That meant the remote path could not adapt candidate choice to arbitrary codebases well enough, even though the user explicitly required an API-driven selection step.
- Progress reporting also still reflected the older 4-step flow, which no longer matched the intended candidate-pool plus selection pipeline.

## What Was Done

- Added dedicated preprocess candidate-selection request and response contracts.
- Added a new remote provider mode, `preprocess-select`, with JSON-only response handling and normalized `selectedTerms` parsing.
- Added local fallback support for preprocess candidate selection so the extension remains resilient when the remote selector is unavailable.
- Reworked file preprocessing to build a raw candidate pool first, then call the provider to choose worth-preprocessing terms, then batch-generate short wordbook entries only for the selected terms.
- Updated preprocess progress reporting from 4 steps to 5 steps: prepare candidate pool, select wordbook terms, send batch request, save cache, complete.
- Rewired the extension entry point so it no longer preselects candidates locally before calling the preprocess builder.
- Cleaned local fallback strings that would otherwise surface corrupted Chinese text when local fallback paths are used.
- Added regression tests for remote candidate-selection normalization and the new preprocess-cache flow.

## Outcome

File wordbook preprocessing now follows the intended product behavior: the raw candidate list is built locally, but the actual selection on the remote path is decided by the configured model using full-file context and audience profile. The old fixed-term filtering remains only as local fallback behavior instead of silently controlling the remote path.
