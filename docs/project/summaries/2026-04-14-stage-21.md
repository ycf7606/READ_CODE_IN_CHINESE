# Stage 21 Summary

## Milestone

- Stage: 21
- Name: LSP-aware wordbook structure
- Date: 2026-04-14

## Delivered

- Rebuilt glossary generation as a hybrid pipeline:
  - regex extraction still covers imports, qualified API calls, labels, and fallback tokens
  - VS Code `DocumentSymbol` data now enriches local class/function/variable/type extraction
  - glossary cache now uses `builderVersion` invalidation so upgraded extraction is actually reused
- Added a pure `documentStructure.ts` layer so document-symbol structure can be tested without loading the VS Code runtime
- Propagated `symbolOrigin` and `scopePath` through:
  - glossary entries
  - preprocess candidate pool
  - remote/local provider normalization
  - preprocess cache reuse
- Upgraded the wordbook tab UI:
  - separate `File Symbols` and `Library / API Symbols` layers
  - term search
  - `All scopes` / `Current class` / `Current function` filtering
  - per-file persisted expand state, active tab, search value, and filter value via webview state
- Strengthened basic markdown rendering with additional inline emphasis support and heading-style line handling
