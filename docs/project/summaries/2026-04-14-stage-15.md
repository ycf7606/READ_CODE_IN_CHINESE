# Stage 15 Summary

## Milestone

- Stage: 15
- Date: 2026-04-14
- Status: Completed

## Root Cause

- Stage 14 added audience-aware wordbook selection, but the default medium profile still needed to be stricter for common users: overly common terms such as `forward` should already be excluded at that level.
- Runtime explanation generation also still depended too much on the generated editable prompt for audience tone, which meant a user profile change could be under-reflected if the prompt was not regenerated.

## What Was Done

- Tightened `professionalLevel = intermediate` so the default medium wordbook profile skips trivial common symbols such as `forward`.
- Kept `beginner` broader and `expert` stricter than `intermediate`.
- Added `occupation` directly to `ExplanationRequest`.
- Injected `occupation` and `professionalLevel` explicitly into runtime explain and follow-up prompts.
- Added audience guidance text in prompt builders so wording can adapt to students, developers, data scientists, researchers, and maintainers while remaining concise and dictionary-like.
- Updated compatibility code paths such as token knowledge prebuild to match the extended request contract.
- Added regression assertions and smoke inspection for both medium-default filtering and audience-aware prompt content.

## Validation

- `npm.cmd run compile`
- `npm.cmd test`
- Smoke inspection through built modules confirmed:
  - `intermediate` candidates excluded `forward`
  - token explain prompts included `Occupation: data-scientist`
  - token explain prompts included `Professional level: intermediate`

## Outcome

The default medium user profile is now closer to the intended product behavior, and explanation generation is more reliably audience-aware because occupation and professional level are injected directly into runtime prompts instead of relying only on regenerated global prompt text.
