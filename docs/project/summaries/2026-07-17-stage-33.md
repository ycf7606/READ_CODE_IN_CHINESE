# Stage 33 Summary

## Milestone

- Stage: 33
- Name: Privacy-safe cache and preprocessing correctness
- Date: 2026-07-17

## Goal

- Prevent repeated-token explanations from crossing library or local callsite boundaries.
- Stop background preprocessing from sending code without an explicit mode and safety policy.
- Invalidate work immediately when unsaved document content or build inputs change.
- Preserve meaningful function scope while sharing context budgets fairly across candidates.
- Add a real VS Code Extension Host regression path before pushing the rebuilt interaction stack.

## What Changed

- Added v2 token knowledge identities using origin, qualified name, and local callsite hashes, with legacy term-only read compatibility.
- Added `off`, `manual`, `onSave`, and `idle` preprocessing modes; `manual` is now the privacy-first default.
- Added workspace trust enforcement, sensitive-file exclusion globs, UTF-8 file-size limits, and candidate-count caps before remote preprocessing.
- Added active-document change cancellation and captured-document-version checks so unsaved edits cannot finish stale preprocess work.
- Added preprocess builder versions and provider/fallback/model/audience fingerprints to prevent incompatible cache reuse.
- Expanded symbol context to retain bounded Python indentation scopes and brace-delimited functions/classes, then distribute total context limits across all candidates.
- Added VS Code definition-provider origin hints alongside hover documentation.
- Added `@vscode/test-electron`, a fixture workspace, an Extension Host smoke suite, and CI execution through `xvfb-run`.

## Outcome

- `numpy.asarray`, `cupy.asarray`, and same-named local variables no longer share unsafe token cache entries.
- Remote preprocessing is opt-in by default and blocked before provider creation when workspace, path, size, or trigger policy is unsafe.
- Unsaved edits and configuration/model changes cannot silently reuse stale preprocessing results.
- The project now validates real command registration, selection updates, panel opening, and document edits inside a VS Code Extension Host.
