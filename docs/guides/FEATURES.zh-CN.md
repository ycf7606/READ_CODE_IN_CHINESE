## 按选区类型解释

插件会先识别单个符号的类型，再构造解释请求。

| 选中内容 | 解释重点 |
| --- | --- |
| 变量或常量 | 含义、值来源、转换过程、使用方和状态变化 |
| 函数或方法 | 职责、输入、输出、副作用和关键调用路径 |
| 类或类型 | 角色、状态、构造方式和公开行为 |
| 模块或包 | 用途、导出能力和与当前文件的关系 |
| 库 API 或内置对象 | 完整名称、签名、精简文档依据和当前调用方式 |
| 语句或代码块 | 控制流、数据流和局部影响 |
| 文件 | 文件职责和主要符号 |
| 工作区 | 文件索引和项目级结构 |

插件会尽量解析 Python 别名、点式导入、多行 `from ... import (...)` 和限定调用。它利用 VS Code 的悬停与定义提供器区分工作区代码和已安装库；查文档时不会导入或执行第三方包。

## 解释面板

面板适合常驻在源代码编辑器旁边。

- 跟随源代码选区，不反复抢走编辑器焦点。
- 展示当前文件、选区、符号类型、来源和限定 API 名称。
- 上下文变化时取消过期解释和追问任务。
- 提供暂停/恢复、重新生成、设置和推理强度控制。
- 当前解释未就绪时禁用追问操作。
- 通过安全的文本 DOM 构造展示模型内容。

## 文件词典与术语表

两个视图分别解决不同问题：

- Explorer 中的 **Code Glossary** 保存可编辑的文件级术语。
- 面板中的 **Wordbook** 展示当前文件已经预处理的符号解释。

文件词典会显示当前文件的完整缓存，并按模块、类和函数作用域分组。`self.load(...)`、`cls.build(...)`、`this.render(...)` 等成员调用也可以识别为文件内函数。

详细流程见[预处理与缓存](PREPROCESSING.zh-CN.md)。

## 知识功能

- 导入本地 `.md`、`.txt` 和 `.json` 文档。
- 为解释和追问检索最相关的工作区知识片段。
- 同步 TypeScript、JavaScript、Python、Go、Rust 和 Java 的预设参考文档。
- 某个文档同步失败时继续保留其他成功结果。
- 在 `.read-code-in-chinese/reports/` 下生成 Markdown 工作区索引。

文档格式见[导入知识文档](../knowledge/IMPORTING_KNOWLEDGE.zh-CN.md)。

## 模型使用体验

- **Local**：无需 API Key，适合离线使用，采用确定性的本地启发式逻辑。
- **OpenAI-compatible**：提供远程结构化解释、候选选择、提示词生成和备用端点。
- 可要求不受信任的工作区始终使用本地模式。
- **Read Code In Chinese** 输出通道会记录运行诊断。

模型设置见[配置说明](CONFIGURATION.zh-CN.md)。
