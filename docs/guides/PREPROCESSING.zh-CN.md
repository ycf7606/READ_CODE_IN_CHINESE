# 预处理与缓存

[English](PREPROCESSING.md) · [简体中文](PREPROCESSING.zh-CN.md) · [文档中心](../README.zh-CN.md)

文件预处理会为用户定义的变量、函数、类、类型和标签类词语生成精简、可复用的文件词典。它优先提取相关上下文，不会简单地把整个文件全部发送出去。

## 触发与隐私策略

| 设置 | 默认值 | 含义 |
| --- | ---: | --- |
| `readCodeInChinese.preprocess.mode` | `manual` | `off`、`manual`、`onSave` 或 `idle` |
| `readCodeInChinese.preprocess.exclude` | 敏感/构建文件规则 | 禁止远程预处理的文件 |
| `readCodeInChinese.preprocess.maxFileBytes` | `262144` | 最大 UTF-8 源文件大小 |
| `readCodeInChinese.preprocess.maxCandidates` | `120` | 单次最多分析的候选符号数 |

工作区不受信任、文件过大或路径命中排除规则时，远程预处理会直接拒绝。只有用户主动选择 `onSave` 或 `idle` 后才会运行后台远程预处理；默认模式为手动。

默认排除环境文件、密钥、证书、常见秘密文件、依赖目录、构建产物、Git 数据、压缩 JavaScript 和 Source Map。

## 处理流程

1. 从语法、赋值、调用、文档符号和标签类字符串中提取原始候选。
2. 为每个符号构建有边界的定义与引用窗口，并尽量保留相关的长函数作用域。
3. 候选少于 24 个时使用本地排序，省去一次模型请求。
4. 候选较多时，让已配置模型根据压缩上下文选择值得处理的术语。
5. 根据用户水平保留候选：初学者全部保留，中级约 85%，专家约 70%。
6. 按阅读优先级将符号分成每批约 20 个。
7. 每批完成后只写入真实结果。
8. 根据最近阅读的选区动态调整剩余批次顺序。

变量从含义、来源、转换和使用方解释；函数从职责、输入、输出和关键副作用解释。

## 上下文准确性

- 每个符号只接收有边界的定义和引用视图，不直接使用整个文件。
- 总上下文预算会在候选之间分配，避免前面的符号挤占后续符号。
- 较长的 Python 缩进作用域和花括号作用域会在固定上限内保留。
- 每个符号都有上下文哈希，不相关位置的修改不会强制重建所有条目。
- 未保存编辑会立即取消正在运行的预处理；编辑稳定后，空闲模式可以重新构建。
- 远程批次失败时按批回退，单个失败不会让整个任务丢失。

## 缓存查询与失效

解释单个符号时按以下顺序查询：

1. 兼容的文件预处理缓存。
2. 身份隔离的 Token 知识缓存。
3. 远程或本地模型。

工作区存储结构：

```text
.read-code-in-chinese/
  preprocess/
  token-knowledge/
  glossary/
  knowledge/
  reports/
  workspace-index.json
```

构建器版本或模型/用户画像指纹变化时，文件缓存会失效。指纹包含模型端点、模型名称、职业、专业水平、用户目标和算法版本。必要时，同名 Token 会按来源、限定 API 名和本地调用上下文隔离。

加载缓存时会自动清理旧版本的占位条目。

## 面板进度

面板分别显示候选池数量、选中目标数量、可复用缓存数量和已处理批次。只有真正进入批处理时才显示批次计数；最终文件词典会按作用域展示当前文件的全部缓存。
