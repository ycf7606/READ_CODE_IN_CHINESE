# 运行流程

[English](RUNTIME.md) · [简体中文](RUNTIME.zh-CN.md) · [架构说明](../ARCHITECTURE.zh-CN.md)

## 选区解释

1. 命令或防抖后的选区事件向源编辑器会话请求当前上下文。
2. 会话启动带版本号的新解释任务，并取消上一个任务。
3. 配置优先读取显式 VS Code 设置，再读取开发环境变量默认值。
4. 工作区信任策略可能强制使用本地模型。
5. 加载术语表和兼容缓存。
6. 选区检查器在限定时间内查询悬停和定义信息。
7. 纯分析逻辑判断解释粒度、符号类型、来源、限定名称和精简文档。
8. 根据选区文本、API 身份、签名和附近上下文检索工作区知识。
9. 单符号请求先查询文件预处理缓存，再查询 Token 知识缓存。
10. 缓存未命中时，由本地或远程模型生成结构化解释。
11. 库文档只附加一次，作为精简依据。
12. 写入面板状态前，再次核对任务版本和编辑器上下文。

最后一次校验可以阻止较慢或失败的旧任务覆盖更新后的选区。

## 面板交互

- 打开或点击 Webview 不会替换当前跟踪的源编辑器。
- 自动更新不会反复显示面板或抢走源代码焦点。
- 暂停跟随只停止新的选区请求，不丢弃当前解释。
- 重新生成会针对当前源代码选区创建新任务。
- 追问有独立活动任务；解释上下文变化时会取消旧追问。
- 控件依赖的数据未就绪时保持禁用。

## 文件预处理

1. 创建模型前先检查触发模式和隐私策略。
2. 未保存文档变化会使当前预处理失效。
3. 构建候选符号和有边界的上下文。
4. 小候选池本地排序，大候选池可以交给模型筛选。
5. 选中术语按优先级分批处理。
6. 每个成功批次写入真实的部分结果。
7. 单批远程失败时可以使用本地回退。
8. 发布进度或最终词典前，会话再次确认任务仍然有效。

排序和缓存细节见[预处理与缓存](../guides/PREPROCESSING.zh-CN.md)。

## 模型失败处理

- 按顺序尝试主端点和已配置的备用端点。
- 请求超时、HTTP 错误、响应格式错误和不支持的参数都会写入日志。
- 兼容性重试可以减少可选模型参数。
- 所有远程端点均失败时，解释流程可以返回本地结果。
- 主动取消不会被显示成模型错误。

## 运行时保证

- 每类工作流最多只有一个活动任务。
- 任务版本只会递增。
- 被替换的任务会取消并释放。
- 选区去重避免完全相同的自动请求。
- 编辑器切换和文档修改会使不兼容任务失效。
- 只有最新且仍有效的任务可以更新 Webview。
