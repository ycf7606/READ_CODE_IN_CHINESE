# 缓存与数据流

[English](CACHE_AND_DATA_FLOW.md) · [简体中文](CACHE_AND_DATA_FLOW.zh-CN.md) · [架构说明](../ARCHITECTURE.zh-CN.md)

所有持久化数据都限制在当前工作区内。

## 存储结构

```text
.read-code-in-chinese/
  glossary/          可编辑的文件术语
  knowledge/         导入和同步的知识文档
  preprocess/        文件级符号词典
  token-knowledge/   重复单符号解释的兼容缓存
  reports/           生成的 Markdown 报告
  workspace-index.json
```

## 解释查询顺序

| 优先级 | 来源 | 兼容键 |
| ---: | --- | --- |
| 1 | 文件预处理缓存 | 文件、符号、上下文哈希、构建器/模型/用户画像指纹 |
| 2 | Token 知识缓存 | 语言、来源、限定 API，必要时包含调用位置身份 |
| 3 | 模型 | 当前选区和请求上下文 |

术语表和检索知识用于补充上下文，不作为最终解释缓存。

## 文件预处理缓存

每个条目对应一个被选中的文件符号，保存生成含义和符号上下文哈希。缓存文件还记录：

- 构建器版本；
- 主模型和备用端点；
- 模型名称；
- 职业和专业水平；
- 用户目标；
- 算法指纹。

不相关修改不会影响上下文哈希稳定的条目；但构建指纹不兼容时，即使符号文本没变也会让缓存失效。

部分写入只包含真正完成的条目。旧版本的占位类条目会在加载时删除。

## Token 知识缓存

文件缓存未命中后，Token 缓存继续作为兼容回退。当前身份模型可以避免不同库暴露同名成员时发生串用。仅靠限定名称仍不足时，会加入本地调用上下文。

成功的远程单符号解释可以写入该缓存。

## 知识库

导入的 `.md`、`.txt`、`.json` 文档和同步的参考页面共享同一个工作区知识库。文档会切分并按关键词重合度评分，标题和标签权重高于正文匹配。

根据 `readCodeInChinese.knowledge.topK`，最相关片段会附加到解释和追问请求。

## 数据安全

- 缓存不会跨工作区共享。
- API Key 从环境变量读取，不写入缓存文件。
- 远程预处理在构造请求前执行信任、路径和大小限制。
- 查找文档时不会导入第三方 Python 包。
- 未保存修改会先取消正在运行的预处理，避免持久化过期结果。
